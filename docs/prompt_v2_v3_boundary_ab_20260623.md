## 结论
- 这轮 20 条隐式边界集上，`v2 structured（还原版）` 为 95.0%，`v3` 为 100.0%。
- `v2` 的失败样例是：`整个处理过程都让我有点崩溃` 被误判为 `其他`，说明在没有“投诉/建议”等显式词时，旧版更容易保守兜底。
- 因此 `v3` 的价值同时体现在两点：
  1. **把隐式语义还原正式写进指令规范**，而不是只靠 few-shot 暗示；
  2. **在新增边界集上实现了从 95.0% 到 100.0% 的提升**。

## v3 相比 v2 的关键变化

### 1. 新增 `boundary_conditions` 章节
- 不是只列边界规则，而是明确要求：
  - 当句子里缺少显式分类词时，先还原业务情境，再做分类。

### 2. 新增“潜在语义推理原则”
- `处理 / 售后 / 恢复 / 撤掉其中一部分` 等词，通常隐含退款退货/售后单语境。
- `门牌 / 门号 / 楼栋 / 填错了 / 改地址` 等词，通常隐含物流配送语境。

### 3. 新增“正确推理过程”示例
- 例如：`一笔里能只处理其中一件吗`
  - 先从“一笔”“处理其中一件”还原为工单/订单中的部分处理语境；
  - 再判断这更像售后/退款单中的部分处理，而不是普通商品咨询。

## 适用建议
- 如果只看基础 structured 能力，`v2` 已经具备较高可用性。
- 如果要强调“隐式边界能力、规范完整性、边界条件可解释性、后续维护性”，推荐将 `v3` 作为正式版本沉淀。
